JF Notify: Instant Telegram Notifications for Gravity Forms

WordPress March 2026
TL;DR: JF Notify is a WordPress plugin that hooks into Gravity Forms and sends a Telegram message every time a form entry is submitted. It uses Gravity Forms' gform_after_submission hook, formats the entry data into a readable message, and delivers it via the Telegram Bot API — with retry logic for reliability. Now live at jfnotify.com.

The Problem It Solves

Gravity Forms has a built-in email notification system. For a lot of use cases, that's enough. But email has a response latency problem: depending on how often you check your inbox, a submitted lead might sit unread for hours. For businesses where fast follow-up is a competitive advantage — service inquiries, consultation requests, time-sensitive quote forms — that delay has a real cost.

Telegram notifications solve this cleanly. The Telegram mobile app delivers messages with the same immediacy as SMS, but with richer formatting, threading, and the ability to route notifications to group chats or specific team members. If your team already lives in Telegram (as many distributed teams do), routing form submissions there means notifications land exactly where the conversation about them will happen.

That's the problem JF Notify solves: bridge the gap between a Gravity Forms submission and a Telegram message, reliably and with enough configuration flexibility to be useful beyond the simplest use case.

How It Works

The plugin hooks into Gravity Forms using the gform_after_submission action, which fires after a form entry has been saved to the database. At that point, the entry data is fully available and the form's conditional logic has already been evaluated.

add_action('gform_after_submission', 'jf_notify_handle_submission', 10, 2);

function jf_notify_handle_submission($entry, $form) {
    $settings = jf_notify_get_form_settings($form['id']);

    if (empty($settings['enabled']) || empty($settings['bot_token']) || empty($settings['chat_id'])) {
        return;
    }

    $message = jf_notify_build_message($entry, $form, $settings);
    jf_notify_send_telegram($settings['bot_token'], $settings['chat_id'], $message);
}

The hook receives the full entry array and the form definition. The plugin checks whether notifications are enabled for this specific form, pulls the configured bot token and chat ID, builds the message, and sends it.

Message Formatting

The message builder iterates over the form's fields and maps them to human-readable labels. By default, every visible field is included. The settings page lets you configure which fields appear and in what order — useful for forms with internal tracking fields that don't need to appear in the notification.

function jf_notify_build_message($entry, $form, $settings) {
    $lines = [];
    $lines[] = "*New submission: {$form['title']}*";
    $lines[] = "";

    $field_map = $settings['field_map'] ?? [];

    foreach ($form['fields'] as $field) {
        // Skip hidden, admin-only, and explicitly excluded fields
        if ($field->visibility === 'hidden' || $field->adminOnly) {
            continue;
        }
        if (in_array($field->id, $settings['excluded_fields'] ?? [])) {
            continue;
        }

        $label = !empty($field_map[$field->id]) ? $field_map[$field->id] : $field->label;
        $value = rgar($entry, $field->id);

        if (!empty($value)) {
            $lines[] = "*{$label}:* {$value}";
        }
    }

    $lines[] = "";
    $lines[] = "_Submitted: " . date('M j, Y g:i a', strtotime($entry['date_created'])) . "_";

    return implode("\n", $lines);
}

The output uses Telegram's MarkdownV2 formatting — asterisks for bold field labels, underscores for the italic timestamp footer. The result is a notification that's readable at a glance without needing to open the WordPress admin to see what was submitted.

Delivery and Retry Logic

The Telegram Bot API's sendMessage endpoint is straightforward, but delivery can fail — temporary network issues, Telegram API rate limiting, or a misconfigured bot token. A failed notification that silently disappears is worse than no notification at all.

function jf_notify_send_telegram($bot_token, $chat_id, $message, $attempt = 1) {
    $api_url = "https://api.telegram.org/bot{$bot_token}/sendMessage";

    $response = wp_remote_post($api_url, [
        'body' => [
            'chat_id'    => $chat_id,
            'text'       => $message,
            'parse_mode' => 'Markdown',
        ],
        'timeout' => 10,
    ]);

    if (is_wp_error($response) || wp_remote_retrieve_response_code($response) !== 200) {
        if ($attempt < 3) {
            // Schedule a retry via WP-Cron with exponential backoff
            $delay = pow(2, $attempt) * 60; // 2 min, 4 min
            wp_schedule_single_event(time() + $delay, 'jf_notify_retry', [
                $bot_token, $chat_id, $message, $attempt + 1
            ]);
        } else {
            jf_notify_log_failure($message, $response);
        }
    }
}

On failure, the plugin schedules a WP-Cron retry with exponential backoff — two minutes for the first retry, four minutes for the second. After three failed attempts, the failure is logged to a custom database table so you can see what was missed without losing the data. The entry itself is always saved to the Gravity Forms database regardless of notification delivery status.

Per-Form Configuration

JF Notify installs settings at the form level, not globally. This means you can configure different Telegram destinations for different forms — a sales form that notifies the sales team's group chat, a support form that routes to a different bot entirely, an internal form that sends to a personal chat. Each form's notification settings are stored in the form's meta via update_form_meta().

The settings panel lives on the form's Settings page in the Gravity Forms admin, following Gravity Forms' own UI conventions so it feels like a native feature rather than a bolt-on plugin.

Conditional Notifications

Some forms only warrant a notification under specific conditions — for example, a quote request form where you only want a Telegram notification when the requested budget exceeds a threshold, or a contact form where notifications should only fire when the inquiry type is "Urgent."

JF Notify evaluates Gravity Forms' own conditional logic rules before sending. The plugin uses GFFormsModel::is_entry_approved() combined with the form's notification conditions to determine whether a given entry should trigger a Telegram message. This means the conditional logic you already configure in Gravity Forms controls JF Notify's behavior — no separate condition configuration needed in the plugin itself.

The Rebrand: From Telegram Notifier to JF Notify

The plugin launched under the name "Telegram Notifier for Gravity Forms." Functional, descriptive, but completely generic — nothing that connects it to the rest of the JF product family or to josefresco.com.

The rename to JF Notify was primarily about building a recognizable brand across products. JF Website Monitor, JF Notify — the naming pattern makes it clear these tools come from the same developer and are built to the same standards. It also gives the plugin a home at a memorable domain (jfnotify.com) instead of relying entirely on the GitHub repository for discoverability.

The rebrand involved updating the plugin's internal text domain, the settings page heading, and the GitHub repository README, plus building the jfnotify.com marketing site as a dedicated landing page.

What's Next

The current version handles the core use case well, but there are two features on the roadmap worth calling out:

  • Multi-chat routing — Send different field subsets to different Telegram chats from a single form submission. Useful for forms where some data should go to the sales team and other data to fulfillment.
  • Delivery receipts — A lightweight log in the WordPress admin that shows which submissions triggered notifications and whether they delivered successfully, without requiring you to dig into the failure log.

Both are scoped and on the backlog. In the meantime, the plugin is stable and handling production traffic on several client sites.
